Installation in networks

The following comments are for those that want to install EUKLID DynaGeo on a network.


Basics:

It is strongly recommended to install EUKLID DynaGeo on the server. Even less experienced administrators are to resist the temptation to install the program on every client locally! A correct network installation is not much more complicated than a local one, but it is much easier to maintain.

To install EUKLID DynaGeo on a network the following steps are necessary:

  • Log in as network administrator.
  • Start the installation archive (e.g. EDG*.EXE) and
  • install EUKLID DynaGeo in a directory of your choice on the server.
  • On every client computer, create a link to the file DYNAGEO.EXE on the server.
  • Make sure that all EUKLID DynaGeo users have the right to read the files in the EUKLID DynaGeo directory. Normal users must not have write access here.


After the install program has done its work, it calls the new DYNAGEO.EXE in the EUKLID DynaGeo directory. This first program start must be done with write access to the EUKLID DynaGeo directory, but this is guaranteed when you're logged in with administrator rights. Now DYNAGEO.EXE will create the common configuration file DYNAGEO.INI. Then it will search for a license data file (*.DGL file), namely in the directory that contains the installation archive. If a DGL file is found there, it will be copied to the EUKLID DynaGeo directory. (If no DGL file is found, the installed version is shareware and so the educational use is prohibited!) Now, the installation of EUKLID DynaGeo on the server is already done.

On the client computer from which you started the installation, a link to DYNAGEO.EXE is added to the start menu. This allows to start the program on this client. If you want to start the program from another client you only need to create a corresponding link on that client, too. It doesn't matter if the link is in the start menu or on the desktop. That's all! EUKLID DynaGeo needs neither any special DLLs in the local windows directory nor any entries in the registry.

To allow your network users to use the freshly installed EUKLID DynaGeo program, you must grant them the rights to read, search and execute files in the EUKLID DynaGeo directory on the server. These rights ought to be extended to the subdirectories too. So every user may have a look at the installed example files. In order to guarantee a high degree of operational safety, the "normal users" are to have no write access to the EUKLID DynaGeo directory and its subdirectories!

Thus, the installation is done, and your users can work with EUKLID DynaGeo. In case of an update, you only need to update the files in the EUKLID DynaGeo directory on the server. No further work at the client computers is necessary!


Some technical details:

For a normal program start of a licensed version of EUKLID DynaGeo the following 4 files are needed:

  • DYNAGEO.EXE,
  • DYNAGEO.INI,
  • DYNAGEO.CHM and
  • a license file *.DGL.
These 4 files must be located in the same directory. If there is no license file EUKLID DynaGeo will run as shareware version.

To enable a user to save his private configuration EUKLID DynaGeo needs one more file. The private configuration file is named EDG_USER.INI, and the user needs write access to it. With older WINDOWS versions (95, 98, ME) this file is located in the EUKLID DynaGeo directory too. With the newer WINDOWS versions (NT4, 2000, XP,...) it is written into the profile of the user, namely to the directory %appdata%\dynageo\. You don't have to bother about that file - EUKLID DynaGeo will manage it all by itself. When the program closes, the actual configuration data are saved to this file - unless the administrator has disabled this option. At the next start, the last saved configuration data are restored again.

It does not matter where the data files (i.e. the *.GEO- and *.MAK files) are located. Every user can write his data on any network drive that is accessible to him.


Please note:

    It's true that for normal use EUKLID DynaGeo needs neither any DLLs in the local WINDOWS directory nor any entries in the registry. But if you want to use the DynaGeoX-Viewer to display dynamic drawings in HTML documents, then some entries in the registry are unavoidable. Fortunately, they are set automatically at the first start of the DynaGeoX viewer!
