Graph of a function

Graph of a function    

Calling this command will open up a small dialogue, where you can enter the term of your the function. Finally you click the dialogue's "Okay" button. DynaGeo closes the term dialogue, creates an appropriate function object and draws the graph of your function into the cooordinate system of the drawing.


Please note:

  1. Inside the term you may use all commonly known standard functions. Moreover, the term can hold references to objects of your drawing. For more details see the page The syntax of terms and the following pages.

  2. Although DynaGeo always calculates the width of angles in radian units, you can enter an angle value in degree units too: just add a degree sign (°) after the angle width. This will cause an automatic conversion from degree units to radian units, whenever the term value is calculated. For more details see: The problem of the angle unit.

  3. The drawing interval of the graph is always the whole actually visible interval of the x-axis.. If the range of the function covers only parts of this interval, the graph is only drawn where DynaGeo is able to calculate valid values of the function term.

  4. The context menu of a function graph shows the following additional menu items:

    • Colour...:
        here, you can select a new graph colour;

    • Line style...:
        here, you can choose an other line style;
        this item may be missing with some functions (see: About functions and graphs).

    • Edit function term:
        here, you can edit the function term later;

    • Show table of values:
        this will call up a window presenting a table of values for this function. The x-values are equidistant from one another and cover a freely choosable range. The default setting is that the x-values cover the actually visible range of the x-axis, with a step width of 1. But you can change the borders of the range as well as the step with. You can even add further columns for values of other functions in your drawing.
        This table window has itself a context menu, that among other things allows to copy its contents into the Windows clipboard. There are some more items, but they are self-explaining.
