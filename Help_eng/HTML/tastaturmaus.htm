Keyboard and Mouse

As for the most WINDOWS applications, a mouse is absolutely essential for EUKLID DynaGeo. The program can virtually completely be controlled via the mouse. The keyboard is only used to enter texts (e.g. object names, comment texts or macro help texts). However, menu commands can also be started by hot keys via the keyboard, which often may actually be faster than clicking with the mouse.

  • The left mouse button serves, as usual, for choosing menu options, tools from the toolbar as well as scaling and moving the EUKLID DynaGeo window. Additionally it can be used to choose objects, drag basis objects and make a selection frame to export part of the drawing.

  • A double click of the left mouse button on an object opens a window in which the object may be given a new name. (See also Edit | Name object). A double click on a comment text brings up a text editor window.

  • A right mouse button click brings up the clicked object's context menue. There you can adjust some object parameters like name, colour and style. For further details see the particular object's description. If you right-click on an empty spot of your drawing rather than an object, you will get the drawing's context menue.


(As WINDOWS allows the right and left buttons to be swapped, the above conventions are only valid for right-handed people. Left-handers will have to swap the buttons.)
