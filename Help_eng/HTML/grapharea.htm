Area below a function graph    

This command draws an "integral area".

    The following description assumes that the "upper border function" runs completely above the "lower border function" resp. above the x-axis. It is clear that DynaGeo can also draw integral areas that do not fullfill this precondition. So this is not a restriction of the program, but serves only to keep the following description simple.
To define the wanted area, please follow these 3 steps:

  1. First, click on the function graph that is to serve as the "upper border".

  2. Second, mark the interval of integration: indicate two points the x-coordinates of which define the interval of integration. You can use already existing points, or create new basis points by a mouse click. If the two points are free basis points, they are snapped to the x-axis.

  3. Finally, the lower border of the area must be specified. There are two possibilities:

    1. A simple "area below a graph" has the x-axis as lower border.
      In this case, click on the x-axis.

    2. If you want to define an "area between two graphs",
      then click on a second function graph.
After that, the appropriate integral area object is created and displayed.


Please note:
  • The left and right border of an integral area is always determined by line segments orthogonal to the x-axis. The upper border is always given by a function graph, while the lower border can be delivered by the x-axis or by a second function graph.

  • To use the size value of such an integral area in a term, there is a standard function area(). If the integral area has the name "Int1", then the term "area(Int1)" delivers the actual size of this area. But please note, that this value gives the so-called "oriented size", which is delivered by the usual integration: so the area between a full period of the sine curve and the x-axis has the oriented size 0 !

  • As long as the border functions are continuous and defined for all real numbers, the calculation of integral areas is done without problems. But if there are gaps in the function's range, then the interval of integration must not contain any of these gaps. If you do not take care for this, the integral area may disappear from your drawing, or the "area()" function will deliver wrong values. E.g., when you use a fractional rational function as border, then the interval of integration must not contain any of the poles of this border function.

  • A special case is given by discontinuous border functions with limited jumps like e.g. the "floor()" function. DynaGeo tries to calculate the corresponding integral areas too, but the results are not as accurate as in the case of continuous border functions, especially when the interval of integration is not completely inside the visible window.


ALT command: Riemann sum

    When you select the command Area below a function graph on the Curves bar and you hold down the ALT key while clicking the button, then instead of the integral area a Riemann sum is created. This needs some additional input informations. The details are described in the section about the command Riemann sum.
