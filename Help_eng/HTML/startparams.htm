Command line parameters

Starting DynaGeo with command line parameters


Basics

WINDOWS can pass "command line parameters" to a starting program. These are short text commands which are added after the name of the executable file. To start a program with parameters you can call it from the command line, e.g. by the menu item Start | Run... in the start menu: a window appears with a single line where you can enter the program file's full path followed by the desired parameters.

Alternatively, you may edit a link to this program: click "Properties" in the context menu of the link and add the parameters to the program call there. But please note that then the program will always be called with these parameters. If you want to apply the parameters only once, you should prefer the above described method using the start menu.


Possible parameters

EUKLID DynaGeo knows the following command line parameters:

  1. the name resp. the full path of a GEO file
    The specified file will be loaded automatically when EUKLID DynaGeo starts. This makes it possible that when you double-click a GEO file in the WINDOWS explorer EUKLID DynaGeo is startet and the file is loaded. However this will work only if your operating system has properly registered EUKLID DynaGeo as an editor for GEO files.

  2. "/debug"
    This parameter (without the quotation marks) ought to be applied only in special cases of persistent problems. If this parameter is passed to the program, a log file named hhdebug.txt is created. Some details about your local WINDOWS system are reported there and several run time events are logged. The file is written to your operating system's temp directory (e.g. "c:\windows\temp\").
