Standard locus lines

Standard locus lines are a special kind of dynamic locus lines.

One drawback of the dynamic locus lines is that they cannot be intersected with other lines. EUKLID DynaGeo's programmer was frightened by the horror vision that some user would try to intersect the locus line of the full tan function with the one of the logarithmic helix ;-)

On the other hand, in some situation you may want to intersect at least some simple locus lines with another line. To achieve this EUKLID DynaGeo implements the "standard mode" for locus lines: if a dynamic locus line obviously is a straight line or it is a circle, then the context menu of the locus line contains an item "Standard curve". If you select this item, then a standard curve object (i.e. a straight line object resp. a circle object) is created as a "child" of the locus line. In the following, instead of the locus line's Bezier approximation this standard curve is shown. If you want EUKLID DynaGeo to convert every new created locus line to a standard curve if possible, then go to Options | Locus lines and activate the appropriate option.

Please note the following problems:

  • The standard curve may be too big.
    The standard curve may hold more points than the original locus line did. Surely every position of the generator point lies on the standard curve, but conversely not every point of the standard curve must be a possible position of the generator point! So the standard curve may not be the locus line, but only include it!

  • The locus line may be a standard curve purely by chance.
    Even if during recording all of the generator point's positions lie on a standard curve, this may be true only for a special configuration of your construction. After dragging the drawing the locus line may change its shape and no longer be a standard curve. So after every drag action all standard locus lines are checked if they still meet the demands. If a standard locus line fails this test, it is converted back to a normal dynamic locus line by invalidating the appropriate standard line object. Any further objects derived from the former standard line become invalid too.
