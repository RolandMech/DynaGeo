How to integrate DynaGeoJ in a HTML page

Normally, it will not be necessary that you yourself create the HTML code described in the following section. EUKLID DynaGeo will do this job when you choose the menu command Save as DynaGeoJ HTML page.... But to benefit from this automatically produced code you need to understand it.

Suppose you created a drawing with EUKLID DynaGeo and saved it in GEO format, e.g. to the file D:\MyData\DynaGeo\GeoFiles\example.geo. If now you want to embed this dynamic drawing in a HTML page, you have to insert the following "<APPLET>...</APPLET>" declaration into the source text of this page:

    <APPLET

      code="applet.DynaGeoJ"
      archive="../dynageoj.jar"
      width=501
      height=330
    >
    <param name="geofile" value="example.geo">
    </APPLET>

Some details about the parameters:

  • "code" contains an unique identification of the entry point in the DynaGeoJ-Applet.

  • "archive" holds a path to the JAR file containing the DynaGeoJ applet: either you use a relative local path that leads from the HTML file to the JAR file (when both files are on the same server) - or a full Internet address where the DynaGeoJ-Viewer can be downloaded (e.g. "http://www.dynageo.de/download/dynageoj.jar").

  • "width" and "height" specify the size (in pixels) of the drawing's display window.

  • "param" holds the name of the GEO file containing your drawing:
    "name" must have the value "geofile" while "value" is set to the GEO file's name (without a path!). To enable the IE to find the GEO file it must be in the same directory as the HTML file itself!


Further hints:

  • If you edit the HTML document later, you should take care to obey some syntax rules. It must be asured that DynaGeo can always recognize the complete <Applet> tag:

    • Upper case and lower case are not important for tag names, but may be very important for file names: even if WINDOWS does not distinguish between upper and lower case in file names, there are many LINUX servers in the web that run case-sensitive file systems. So be careful if you want to place your HTML file in the web!

    • Take care that the tags "<APPLET..." and "</APPLET>" always stand alone in an own line of the HTML text.

    The best way is to leave the "<APPLET>" tags untouched and in the state they were created by DynaGeo!

  • In a newly created DynaGeoJ HTML page, the file name of the HTML document and the file name of the drawing are the same and differ only in the extension. It is recommended to stick to this strong synchronisation and to be content with one DynaGeoJ applet per HTML page. It is possible to show more than one GEO file in one HTML page by adding several <Applet> tags that point to different GEO files. But in some cases this may lead to trouble.

  • If you use a full internet address in the "archive" tag, you get a freely movable HTML document. If you use a relative path pointing to a local JAR file, this is no longer valid on another computer. Nevertheless, it is recommended to use a relative path from the HTML document to a local copy of the JAR file when publishing HTML pages on your own web server because in this case, your page will not depend on external ressources!
