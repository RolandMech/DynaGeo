About deleting objects

When you delete an object it is no longer visible on the screen. However, it is not yet physically deleted, but saved internally in case you regret it was deleted. Actually, such an object internally behaves very much like a hidden object. It is not visible on the screen, and you cannot use it as a parent to construct other objects. It is only marked to be deleted soon.

Does EUKLID DynaGeo keep remembering all these delete-marked objects forever?

Of course not. As long as you do not save your drawing, you have the possibility to revive all delete-marked objects by the command Redo last undo. But if there are many delete-marked objects in your drawing, they will take up space in memory and slow down the program. To avoid this, just save your drawing: while writing the GEO file, DynaGeo will physically delete all those delete-marked objects. But keep in mind that then these objects are lost forever.
