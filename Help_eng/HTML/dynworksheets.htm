How to produce dynamic worksheets

The easiest way is to use one of the following menu commands:

  • Save as DynaGeoX HTML page
  • Save as DynaGeoJ HTML page
Whenver EUKLID DynaGeo executes one of these commands, two files of the same name but different extensions are written to the target directory:
  • The first is a HTML file describing the window in which the drawing should appear.
  • The second is a GEO file containing the drawing that should be shown.

Embedding a dynamic drawing in a HTML page is done by the help of special tags: with DynaGeoX, an <OBJECT> tag is used, while DynaGeoJ uses an <APPLET> tag. You can edit the HTML file with a suitable editor, but you must take great care that the <OBJECT>- resp. <APPLET> tag remain unaltered. Some HTML editors are rather inconsiderate towards things they don't understand...

Of course you can copy the <OBJECT>- resp. <APPLET> tag into another HTML document. So you can create pages with more than one drawing embedded. But please take care that at runtime the corresponding GEO files are available together with the HTML file in one directory!

Though the task is highly automated by the corresponding menu commands, you ought to act with some caution to assure that the results achieved meet your expectations. In any case please take care for the following items:

  • Do not choose the viewer window too large! It ought to be displayed completely inside the browser window. On the other hand, make it large enough to show all relevant objects of your drawing, at least at the start.

  • If ever possible, don't use text boxes inside your drawing! They occupy too much of the valuable space of the viewer window. You'd better put your comments and hints in the surrounding HTML page, next to the viewer window.
DynaGeoX uses the same data structures as DynaGeo itself. So normally a drawing that works as expected when loaded into DynaGeo, will also work correctly in the DynaGeoX viewer. However, the export to DynaGeoJ needs some more attention because there are a lot of internal differences between DynaGeo and DynaGeoJ.
