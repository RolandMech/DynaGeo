Save as DynaGeoJ HTML page...

This commands lets you export your drawing into a DynaGeoJ HTML document. When you load this HTML document into an Internet browser then the DynaGeoJ viewer cares for the representation of your drawing. Internal Details about this procedure can be found in the section Dynamic drawings in HTML documents.

When you select this command, DynaGeo first checks if the viewer knows all of the objects of your drawing. There is a small number of DynaGeo objects not known to DynaGeoJ (yet). If such objects are found, you get a list of the unexportable objects and you are asked if you want to export the drawing nevertheless. If you refuse, the export procedure is canceled. But when DynaGeoJ knows all objects of your drawing, or you decide to export only those that are known to it, then a dialog window appears where you can specify the details of the HTML page to build:

  • Target path of the HTML file:
    Initially this field shows the full path to your current drawing, but with the extension ".html" instead of ".geo". You can edit this path as you like, but the extension needs to be ".html" (oder ".htm").
    When you enter an existing HTML file, then DynaGeo will examine the content of this file. If it contains a DynaGeoJ object that points to another GEO file than the one you want to export now, or if the HTML file holds even more than one DynaGeoJ objects, then a window "Choose DynaGeoJ object" is shown and you must decide how your actual drawing should be inserted into the HTML document.


  • Path of the DynaGeoJ viewer:
    This path must point to an actual version of the DynaGeoJ Viewer. Normally you find a suitable file in the sub-directory Viewer of your DynaGeo directory. You can choose from two standard paths to the Viewer: the local one has the advantage that the Viewer can be called without any Internet access; on the other hand, the Internet path assures that you always work with the last recent version of the Viewer. Anyhow, this path must point to a JAR archieve that holds the DynaGeoJ viewer. The standard value provided is an URL to the DynaGeo website where you always may find an actual version of DynaGeoJ. Please change this path only if you know very well what you are doing!


  • Text above the DynaGeoJ object:
    Here you may enter any text you like. Hard line breaks (by pressing the ENTER key) are automatically converted to HTML breaks (”<br>”) umgesetzt. You can also use your own HTML tags here.


  • Viewer window:
    Here you can adjust the width and height (in pixels) of the Viewer window. You should not size the window bigger than necessary, to make sure that your HTML page can also be displayed on smaller screens. Whenever you call the command "Save as DynaGeoJ HTML page" the width and size of the DynaGeoJ object are preset according to the following rules:

    • If you had drawn a selection frame before, then its width and height are taken as preset values for the DynageoJ object size.
    • If there is no selection frame, but the target HTML file exists already, then DynaGeo tries to take over the values of width and height from this file.
    • If the target file does not exist yet, and the DynaGeo program does not appear maximized on the screen, then the width and height of the actual drawing window are taken as preset values.
    • In any other case the width is set to 500 pixels and the height to 350 pixels.

    So it is appropriate to draw a selection frame before calling the command "Save as DynaGeoJ HTML page". Alternatively, you can also resize the DynaGeo window itself in order to let it show just the right part of your drawing. Then it is not necessary to change the size values manually.


  • Viewer commands:
    A click on this button opens a window, where you can choose from a list of commands which of them you want to be available in the viewer window. At runtime, the viewer will display a tool button for every selected command. These tool buttons will be placed in a vertical panel on the left side of the viewer window.
    When no commands are selected, this tool panel is hidden and the whole viewer window space is available to your drawing. When on the other hand you try to select too much commands and not all tool buttons find place in the the tool panel, then you get an appropriate warning.


  • Author's name:
    Whatever you enter here will be written to an appropriate "META" tag in the "HEAD" section of your HTML document. If you leave this field empty, then no such "META" tag will be created. But if you want to publish your document in the Internet it's a matter of "netiquette" to sign it with your name.

When all options have been set to the desired values, a click on the button Save will create the HTML file containing a complete HTML page with a DynaGeoJ object tag. If the file already exists, only the new information is added to the existing file.
Additionally, the actual GEO file is saved too, to update it and register the selected viewer commands if necessary.
