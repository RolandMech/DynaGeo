How to integrate DynaGeoX in a HTML page

Normally, it will not be necessary that you yourself create the HTML code described in the following section. EUKLID DynaGeo will do this job when you choose the menu command Save as DynaGeoX HTML page.... But to benefit from this automatically produced code you need to understand it.

Suppose you created a drawing with EUKLID DynaGeo and saved it in GEOX format, e.g. to the file C:\Programme\DynaGeo\Daten\example.geox. If now you want to embed this dynamic drawing in a HTML page, you have to insert the following "<OBJECT>...</OBJECT>" declaration into the source text of this page : 

   <OBJECT
       classid="clsid:2EF98DE5-183F-11D4-83EC-EC6A1DB6E213"
       codebase="http://www.dynageo.com/download/dynageoviewer.cab#version=2,5,1,0"
       width=501
       height=330
   >
   <param name="DataFile" value="example.geox">
   </OBJECT>


Some details about the parameters:

  • "classid" contains the unique identification number of the DynaGeoX component. This number is set by the programmer of DynaGeoX and must not be changed.
  • "codebase" contains the internet address from where the DynaGeoX viewer can be downloaded. There is a version info too: the embedded drawing can be displayed by the indicated version of the viewer or a newer one. In most cases, an older viewer version will not do!
  • "width" and "height" specify the size (in pixels) of the drawing's display window.
  • "param" holds the name of the GEOX file containing your drawing:
    "name" must have the value "DataFile" while "value" is set to the GEOX file's name (without a path!). To enable the IE to find the GEOX file it must be in the same directory as the HTML file itself!

To spare the internet access when DynaGeoX is needed for the first time you can also specify a local codebase. If e.g. you copied the file DynaGeoViewer.cab into the directory K:\Public\Utils\DynaGeoX of your local network, the following codebase can be used:

    codebase="file:///k:/public/utils/DynaGeoX/DynaGeoViewer.cab#version=2,5,1,0"
The path must be formulated as an URL. Only then the IE can install (or update) the component automatically.

However, in order to create tranportable documents it is strongly recommended to always use the internet codebase. If you specify a local codebase this path usually becomes invalid when you move your document to another computer. And even with an internet codebase there will be no internet access if a sufficiently actual version of the viewer is already installed on your local computer.
