Geometria: the JAVA way

Geometria is a JAVA program written by Dr. Timo Ehmke. It allows to create interactive HTML pages with dynamic drawings. The abilities of Geometria go far beyond those of the DynaGeoX viewer. So Geometria allows to create "worksheets with feedback" by specifying reactions to the user's input.

Dr. Ehmke supplies the Geometria program as freeware. For further details see his Geometria internet pages at http://www.geometria.de/. There you can find a detailed documentation, some examples and a link to the pages of the "Institut für Mathematik der Universität Flensburg" where you can visit a great gallery of geometric problems illustrated with the help of Geometria.


Before you use EUKLID DynaGeo to create HTML pages for Geometria, please make sure that an actual JAVA VM is installed on a your computer. Then download "geometria.jar" from the Geometria homepage and copy it into the subdirectory "viewer" of your EUKLID DynaGeo directory.

Due to the different data structures used in Geometria and EUKLID DynaGeo there are the following incompatibilities:

  • Use only letters and digits in object names. Avoid any special characters even if EUKLID DynaGeo would allow them. Take care not to use names that differ only in capitalization.

  • Geometria does not support segments of fixed length; they are exported as normal segments between two points.

  • Only the distance of a point to another point, a line or a circle can be exported correctly. In the last case, the point has to lie outside the circle.

  • Geometria does not support individual line styles. All lines are displayed as thin solid lines. Only the colour can be exported correctly.

  • Geometria does not support ratio expressions in the same way EUKLID DynaGeo does. Please avoid terms containing ratio expressions.

  • Geometria always puts all the number objects ("slider") at the lower edge of its window, side by side in one line. But if the window is not wide enough then some of the sliders are omitted. In this case you can reduce the sliders' width in the SCRIPT file by hand. For further details see the Geometria documentation.

If a SRIPT file does not work correctly you may debug it. SCRIPT files are normal text files and can be edited with every ASCII editor. Any construction step can be deactivated by inserting "//" in front of the corresponding line. You may now deactivate all of the construction steps but the first few, then test the file again. Now re-activate the construction commands one by one, always testing the file again, until you find the error spot.

However, most of your EUKLID DynaGeo drawings may be exported to the Geometria format without problems.
