File | Save as Geometria HTML Page...

This commands allows to export a drawing to a Geometria HTML document. When you load this document into any JAVA enabled browser, then the JAVA applet Geometria by Timo Ehmke will show your dynamic drawing inside the HTML page.

Please notice:

    The JAVA program "geometria.jar" is not included with EUKLID DynaGeo! It is freely available to all users from the Geometria Homepage. If you download the program and copy "geometria.jar" to the sub-directory "Viewer" of your EUKLID DynaGeo directory, you are prepared to perform the Geometria export in a comfortable way.


Choosing this command will open a dialog box where you can enter the options of the new HTML page:

  • Target path for the HTML file:
    Initially this field shows the full path to your current drawing, but with the extension ".html" instead of ".geo". You can edit this path as you like, but the extension needs to be ".html" (oder ".htm").


  • Copy "geometria.jar" to the target directory:
    This checkbox is activ only if the sub-directory "Viewer" of your EUKLID DynaGeo directory contains a copy of the file "geometria.jar".

    If you check this box, EUKLID DynaGeo will copy "geometria.jar" into the target directory, if necessary. When you leave this box unchecked, you must copy this file manually.


  • Text above the Geoemtria object:
    Here you can enter any desired text. Hard line breaks (by pressing ENTER) are automatically converted to HTML breaks (”<br>”). If you use HTML tags to format your text, these tags will be preserved.


  • Viewer window:
    Here you can adjust the width and height (in pixels) of the Viewer window.


  • Style:
    Here you decide in which style the drawing will be presented: you can choose between a "DynaGeo style" or a "Geometria style".


  • Author's name:
    Whatever you enter here will be written to an appropriate "META" tag in the "HEAD" section of your HTML document. If you leave this field empty, then no such "META" tag will be created. But if you want to publish your document in the Internet it's a matter of "netiquette" to sign it with your name.

When all options have been set to the desired values, a click on the button Save will create two files in the target directory:

  1. a HTML file containing a complete HTML page with a Geometria object tag;

  2. a SCRIPT file containing the drawing data in Geometria format;

  3. a STYLE file containing the presentation details.
The last two files carry the same name as the HTML file, but there extensions are ".script" and ".style" instead of ".html". If possible and desired, a copy of "geometria.jar" is written to the destination directory too.


Never use another STYLE file but the one created by EUKLID DynaGeo! The size of the applet window is written to the STYLE file as well as to the <APPLET> tag in the HTML file. The easiest way to keep the corresponding values synchronized is to leave the whole job to EUKLID DynaGeo.
