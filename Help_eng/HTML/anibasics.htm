The basics of animations

An animation is a controlled change of your drawing's configuration in time. It is started by you, but then it runs independently. It can be interpreted as an automation of the drag mode. The role of the basis object dragged by the mouse is now played by the so-called steering object: in both cases, all dependent object are dragged accordingly.

So if you want to animate your drawing, you must construct it in a way that all objects that are to move dependend on a steering object. Possible steering objects are all objects that can be manipulated internally by the program in a controlled way. At the time being, there are only two types of steering objects:

  1. basis points that are bound to line segments, circles or polygons
  2. number objects
Please note that basis points bound to lines, rays or locus lines can not be used as steering objects. The position of the basis point on its carrier line is described by a position parameter. If the point is to serve as steering object, its position parameter's values must be all inside a finite interval. This is no problem for all the carrier lines of finite length like segments and circles, but (long) lines are "to big" with respect to animations. And in case of locus lines it is a matter of caution to better avoid them: even if a locus line now has finite length, nobody can guarantee if this still holds after the next dragging action.


Basis points as steering objects:

    If you want to run an animation controlled by a bound basis point, you must choose this point as the animation's steering object. The dialogue "Params of animation" lets you choose from a list of possible steering objects. Choose the desired point and close the dialogue. Then the "Animation toolbar" will show the buttons of the Navigation commands that allow you to start and stop the animation.

    If e.g. you click on the green arrow pointing to the right, the animation will start. While it is running the internal position parameter of the steering point is incremented by the step width, then the point's new position and the new positions of all of its "children" are calculated and the drawing is updated. Then the position parameter is incremented once more....

    Normally, the animation stops when the position parameter reaches the border of its preset interval. But if the steering object is a point bound to a closed line then the parameter interval is thought to be "closed" too: when the position parameter reaches one border of its interval, it automatically jumps to the other border and the animation continues. This leads to an endless loop, and such an animation can only be terminated by pressing the "stop button".


Number objects as steering objects:

    The second kind of steering objects are the number objects. With them, the number value itself plays the role of the position paramter. If you want to use a number object N1 as steering object, the objects you want to animate must be dependent from N1. To achieve this, you can use objects with numeric parameters, like (points with determined co-ordinates, circles with determined radii,...). In contrast to the case of basis points now you can choose the bounds of the parameter interval freely: the appropriate edit fields in the dialogue "Parameters of animation" can be edited if a number object is chosen as steering object. The course of the animation works as described above.


The speed of the animation depends directly on the size of the param step: greater steps lead to faster animations, smaller steps result in slower ones. But please note that the animation is not really time-controlled! The same step width can lead to very different speeds of animation. What speed you get depends mainly on the complexity of the animated drawing, but also on the hardware power of your computer!
