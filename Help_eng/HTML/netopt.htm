Network options

This dialogue is called by the menu item Network options in the local menu of the Options window. Because this menu item is enabled only for users with write access to DYNAGEO.INI, this dialogue is accessible for administrators only.

The private options of each individual user are saved in a private file named EDG_USER.INI. With modern network versions of WINDOWS (Win2000, WinXP, Vista,...), this file is saved to the profile of the user (in the directory %appdata%\dynageo\).

For the administrator is a user too, he also has a private configuration data file named EDG_USER.INI. With this dialogue, the suitable options can be transfered from this private configuration file to the common configuration file DYNAGEO.INI. Additionally, the network related rights of the user are managed here.

The dialogue contains the following two sections:


Permission of configuration

    Here the administrator can grant or revoke the rights of the users to manage their DynaGeo configuration on their own. Direct after a new installation, the rights of the users are not restricted at all.

    • If the administrator does not allow the users to edit their private configuration and change their settings, then the menu item Miscellaneous | Options is deactivated for the users, and they must work with the standard configuration.

    • Even if the administrator allows the users to edit their configuration, he still can deny the right to save the modified configuration. So all users at least start with the standard configuration.

    • The menu configurations are a special case:
      If the administrator denies the right to choose another menu configuration, then the user must be content with the standard menu configuration. On the tabsheet Menu configuration of the Options dialogue, he will find the menu commands disabled.

    • Even if the administrator allows the user to change the menu configuration, a user can only create his own menu configuration if he is expressly allowed to do this. Such private menu configurations are saved in the user's private configuration file and thus are invisible to others.

    The options are ordered by decreasing restriction. The mutual dependences of the options are managed automatically.


Management of menu configurations

    This section allows the comfortable administration of the menu configurations. It contains two lists next to each other:

    • the left one shows the actually available private menu configurations of the administrator, contained in his private configuration file (EDG_USER.INI);
    • the right one shows the global menu configurations, contained in the global configuration file (DYNAGEO.INI).

    The two little arrow buttons between the lists allow the administrator to move a selected menu configuration from one list to the other.

    Below the list with the private menu configurations there are three buttons. You can

    • create a new menu configuration
    • edit an existing menu configuration
    • delete one of the existing menu configurations

    These 3 buttons work the same way as the corresponding buttons of the tabsheet Menu configuration of the Option dialogue.

    Below the list of the global menu configuration there is a button which allows the administrator to register the selected global menu configuration as standard menu configuration for all users.



In the whole dialogue, only those commands are enabled that are suitable in the present situation. E.g. the buttons beneath the configuration lists are active only if an corresponding configuration in the list is selected.


Please note:

    Whenever you'll click a button the changed configuration will always be written to the correponding configuration file at once! So take care of what you're doing in this dialogue!
