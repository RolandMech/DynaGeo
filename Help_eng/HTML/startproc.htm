Start procedure

Initialisations at program start

When you start EUKLID DynaGeo the title window appears. It shows the program's name and version, and the license details. While this window is shown, EUKLID DynaGeo checks its own executable file: if there have been made any changes to this file, it is assumed that the file is infected by a virus and the program is terminated with a corresponding message. If you get a virus warning from EUKLID DynaGeo, please take it seriously! In almost all cases, the diagnosis is correct and the system really is infected by a virus.

If the executable file passes the test, the configuration data are read and applied. EUKLID DynaGeo reads these data from two files:

  1. The common standard configuration data are read from the file DYNAGEO.INI in the EUKLID DynaGeo directory. If there is no such file an attempt is made to create it. If this fails the program is terminated with an appropriate message. DYNAGEO.INI is necessary for a correct program start!

  2. The user-specific ("private") configuration data are read from a file named EDG_USER.INI. On computers with a newer version of WINDOWS (2000, XP, Vista,...), it is placed in the directory %appdata%\dynageo\. In networks this directory usually is contained in the users profile or in his %home% directory.

As long as you do not restrict the network options, the program will try to return to the same conditions as existed when it was last ended. This includes the options for the screen display of geometrical objects (point size, capture area, etc.), the menu configuration, the LRU list and the previously loaded macros.


After this procedure is finished successfully, the title window is closed, and the EUKLID DynaGeo window appears.
