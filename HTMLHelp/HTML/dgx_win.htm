DynaGeoX und Windows

Der Normalfall: Automatische Verwaltung von DynaGeoX

    DynaGeoX ist eine ganz gewöhnliche ActiveX-Komponente und wird als solche weitgehend automatisch von Ihrem WINDOWS-System verwaltet. Wenn der Internet Explorer eine HTML-Seite anzeigt, für die eine ActiveX-Komponente benötigt wird, dann wird zuerst bei Windows nachgefragt, ob diese Komponente schon im Windows-System vorhanden ist und - wenn ja - ob in einer hinreichend aktuellen Version. Wenn dies der Fall ist, wird diese lokal schon vorhandene Kopie der Komponente eingesetzt, um das aktive Objekt auf der HTML-Seite darzustellen. Für den anderen Fall enthält die HTML-Seite einen Link zu einer "code base"; dies ist in der Regel eine Internet-Adresse, die angibt, von wo die Komponente heruntergeladen werden kann. Falls nötig, wird also die ActiveX-Komponente von dort geholt, gegebenfalls ausgepackt, in Windows installiert und registriert. Schließlich kann das aktive Objekt auf der HTML-Seite mit dieser neu installierten Kopie der Komponente angezeigt werden.

    Glücklicherweise geschieht die Installation einer aus dem Internet geladenen ActiveX-Komponente durch den Internet Explorer recht transparent: im Windows-Verzeichnis richtet der IE ein Unterverzeichnis namens "Downloaded Program Files" ein, in das die Komponente kopiert wird. Wenn Sie im Explorer in dieses Verzeichnis wechseln und dann dort den Eintrag einer ActiveX-Komponente löschen, dann entfernt der IE auch gleich deren Registrierung aus der Systemdatenbank (Registry). Somit können Sie eine heruntergeladene ActiveX-Komponente auch wieder problemlos aus Ihrem System entfernen. Wenn DynaGeoX auf Ihrem System installiert und registriert ist, dann sollte in diesem Verzeichnis ein entsprechender Eintrag zu finden sein.


Wie Sie eine Installation / ein Update von DynaGeoX auslösen können:

    Der DynaGeoX-Viewer gehört zum Standard-Lieferumfang von EUKLID DynaGeo, so dass er normalerweise gleich bei der Installation ins Betriebssystem eingebunden wird. Trotzdem könnte es sein, dass auf Ihrem Rechner doch kein DynaGeoX installiert ist, etwa weil Sie bei der Installation die Option "DynaGeoX-Viewer installieren" abgewählt haben. Und schließlich kann es passieren, dass Ihre DynaGeoX-Installation veraltet ist. In all diesen Fällen müssen Sie eine Installation bzw. ein Update des DynaGeoX-Viewers manuell anstoßen, wofür es mehrere einfache Möglichkeiten gibt:

    • Im Unterverzeichnis "Viewer" Ihres DynaGeo-Verzeichnisses finden Sie ein HTML-Dokument namens "InstallDGX_InternetSource.html". Wenn Sie diese Datei in Ihren Internet Explorer laden, wird dadurch eine automatische Installation bzw. ein Update von DynaGeoX ausgelöst. Weitere Hinweise (auch für den Fall, dass die Installation nicht klappt) finden Sie in diesem Dokument selbst.

    • Alternativ können Sie DynaGeoX auch durch einen Besuch in der "Galerie" der DynaGeo-Homepage aktualisieren. Wenn Sie dort auf die "Installations-Seite" gehen, wird automatisch überprüft, ob in Ihrem System ein DynaGeoX-Viewer installiert und registriert ist; wenn DynaGeoX nicht installiert ist oder wenn die installierte Version veraltet ist, dann wird ein aktuelles DynaGeoX heruntergeladen und in Ihr Windows-System integriert.

    In diesen beiden Fällen kommt der Viewer per Download auf Ihren Rechner. Bevor Windows ihn dann installiert, werden Sie allerdings nochmals gefragt, ob Sie dies auch wirklich wollen. Je nach Version des installierten InternetExplorers kann es sein, dass Windows die digitale Signatur von DynaGeoX nicht anerkennt und vor einer "nicht vertrauenswürdigen Komponente" warnt. Dann müssen Sie Ihren ganzen Mut zusammennehmen und den Viewer eben trotzdem installieren lassen. Der Vorteil der "Installation per Download" ist aber, dass Sie dabei stets sicher sein können, die neueste DynaGeoX-Version zu bekommen. Diese Methoden sind daher auch geeignet, Ihre DynaGeoX-Installation auf den neuesten Stand zu bringen.

    Eine dritte Möglichkeit ist die lokale Installation:

    • Wenn im Unterverzeichnis "Viewer" Ihres DynaGeo-Verzeichnisses die Datei "dynageox3.ocx" vorhanden ist, dann können Sie auch die ebenfalls dort stehende Datei "InstallDGX_LocalSource.html" in den IE laden. Diese Art der Installation braucht keinen Internet-Zugriff, weil DynaGeoX aus der lokal vorhanden OCX-Datei installiert wird. Der Nachteil ist, dass Sie dabei nicht sicher sein können, ob diese OCX-Datei noch aktuell ist. Daher sollten Sie diese Methode nur dann einsetzen, wenn Sie genau wissen, warum Sie das tun.
