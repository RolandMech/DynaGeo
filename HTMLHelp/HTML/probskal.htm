Probleme mit der Skalierung

a) Skalierung auf dem Bildschirm:

EUKLID DynaGeo bezieht sein Wissen über die Details Ihrer Hardware von Windows. Wenn Windows feststellt, dass da ein 17-Zoll-Monitor angeschlossen ist, der mit 1024 x 768 Pixeln betrieben wird, dann resultiert daraus eine bestimmte Anzahl von Pixeln pro Zentimeter. Diese Zahl erfragt EUKLID DynaGeo von Windows und startet dann so, dass eine Strecke der Länge 5 cm auch genau 5 cm auf dem Bildschirm lang sein sollte.

Da sich jedoch die Bildgröße bei fast allen Monitoren von Hand beeinflussen lässt (und darüberhinaus manche Monitore auch etwas mogeln, wenn Windows nach Daten fragt), muss diese Zahl nicht unbedingt den tatsächlichen Verhältnissen entsprechen. Üblicherweise ist der Bildschirmzentimeter etwa 5% kleiner als er sein sollte. Dies liegt also letztlich daran, dass Windows die Hardware nicht genauer (er)kennen kann.


b) Skalierung beim Ausdrucken:

Auch beim Ausdrucken ist EUKLID DynaGeo auf die Daten von Windows angewiesen. Falls Sie den Skalierungsfaktor auf 1.00 stellen, dann wird die Zeichnung so ausgedruckt, dass EUKLID DynaGeo und Windows glauben, sie im Maßstab 1:1 ausgegeben zu haben. Da Drucker in der Regel etwas ehrlichere Geräte sind als Bildschirme, sollte das meist ziemlich genau stimmen. Bei einem Skalierungsfaktor > 1.00 ( < 1.00 ) wird die Zeichnung entsprechend vergrößert (verkleinert) ausgedruckt.
