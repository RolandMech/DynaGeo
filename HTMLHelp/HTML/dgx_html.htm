Wie wird DynaGeoX in eine HTML-Seite eingebunden?

Normalerweise werden Sie den in diesem Abschnitt beschriebenen HTML-Code nicht selbst erzeugen müssen. Dies erledigt EUKLID DynaGeo für Sie, wenn Sie den Befehl Datei | Als DynaGeoX-HTML-Seite speichern... anwählen. Falls Sie das HTML-Dokument jedoch weiterbearbeiten wollen oder den erzeugten HTML-Code in anderen HTML-Dokumenten weiterverwenden wollen, sollten Sie verstehen, wie DynaGeoX eingebunden wird. Daher wird hier der HTML-Code für die Verwendung von DynaGeoX mit dem Internet Explorer (ab Version 5) beschrieben:

Sie haben mit EUKLID DynaGeo eine Zeichung erstellt und diese in einer GEO-Datei abgespeichert, z.B. unter C:\Programme\DynaGeo\Daten\BEISPIEL.GEO. Wenn Sie nun diese Zeichnung in einer HTML-Seite angezeigt haben wollen, dann müssen Sie an der entsprechenden Stelle des HTML-Quelltextes die folgende "<OBJECT>...</OBJECT>"-Deklaration einfügen:

    <OBJECT

      classid="clsid:2EF98DE5-183F-11D4-83EC-EC6A1DB6E213"
      codebase="http://www.dynageo.de/download/dynageox3.cab#version=3,5,5,0"
      width=501
      height=330
    >
    <param name="DataFile" value="beispiel.geo">
    </OBJECT>

Ein paar Erläuterungen zu den einzelnen Parametern:

  • "classid" enthält eine eindeutige Identifikationsnummer für die DynaGeoX-Komponente.
  • "codebase" enthält die Internet-Adresse, unter der der DynaGeoX-Viewer verfügbar ist. Die Zeile enthält außerdem eine Versions-Information: die eingebundene Zeichnung kann mit der hier angegebenen oder einer neueren Version des DynaGeoX-Viewers angezeigt werden, eventuell (bzw. sehr wahrscheinlich) aber nicht mit einer älteren Version!
  • "width" und "height" geben die Abmessungen des Ausgabefensters (in Pixeln) an.
  • "param" enthält den Namen der GEO-Datei, die die Zeichnung enthält:
    "name" muss hier den Wert "DataFile" haben, und in "value" steht der Name (ohne Pfad!) der anzuzeigenden GEO-Datei. Dabei wird vorausgesetzt, dass sich diese Datei im selben Verzeichnis befindet wie die HTML-Datei.

Um sich den Internet-Zugriff bei der erstmaligen Installation von DynaGeoX zu ersparen (z.B. in Ihrem schulischen Intranet), können Sie auch eine lokale Quelle als "codebase" angeben, z.B.:

    codebase="file:///k:/public/utils/DynaGeoX/dynageox3.cab#version=3,5,5,0"

wenn Sie die Datei dynageox3.cab ins Verzeichnis K:\Public\Utils\DynaGeoX kopiert haben. Damit die automatische Installation des Viewers (oder ein Update!) auch klappt, muss dieser Pfad jedoch als URL formuliert sein.

Im Allgemeinen sollten Sie aber bei allen selbsterstellten DynaGeoX-HTML-Dokumenten für "codebase" den Internet-Pfad verwenden, weil Sie nur so frei distributierbare Dokumente bekommen: ein lokaler Pfad ist natürlich nicht mehr gültig, wenn Sie das Dokument auf einen anderen Rechner transportieren! Und wenn in Ihrem Windows-System schon eine aktuelle Version von DynaGeoX installiert und registriert ist, dann wird trotz der Verwendung des Internet-Pfades zum DynaGeoX-Viewer selbst beim ersten Öffnen eines DynaGeoX-HTML-Dokuments kein Internet-Zugriff benötigt!



Weitere Hinweise:

  • Wenn Sie das HTML-Dokument weiterbearbeiten, müssen Sie ein paar Syntax-Regeln beachten. Damit Sie sich die Möglichkeit einer zukünftigen Überarbeitung der Datei in EUKLID DynaGeo erhalten, muss sichergestellt werden, dass EUKLID DynaGeo die DynaGeoX-Objekte auch stets zuverlässig erkennen kann:
    • Groß- und Kleinschreibung spielt bei den Tag-Namen keine Rolle. Seien Sie aber bei Dateinamen in dieser Beziehung sehr vorsichtig, wenn Sie ihre Dateien ins Netz stellen wollen: im Gegensatz zur Praxis unter WINDOWS ist für LINUX-Server die Groß- und Kleinschreibung bei Pfadangaben signifikant!
    • Achten Sie stets darauf, dass die umschließenden Tags "<OBJECT..." und "</OBJECT>" immer alleine in einer eigenen Zeile des HTML-Quelltextes stehen. Es darf also weder vor dem "<OBJECT..."-Tag noch nach dem "</OBJECT>"-Tag irgend eine andere wesentliche HTML-Information stehen. Lediglich ein Zeilenumbruch ("<BR>") ist erlaubt.
    Am besten belassen Sie die "<OBJECT>"-Tags einfach in dem Zustand, in dem sie von EUKLID DynaGeo erzeugt wurden!

  • Bei neu erzeugten DynaGeoX-HTML-Seiten stimmen üblicherweise der Dateiname des HTML-Dokuments und der Dateiname der referenzierten Zeichnung bis auf die Erweiterung überein. Sie können diese strenge Synchronisation zwischen den Dateinamen aufbrechen, indem Sie im Fenster "Als DynaGeoX-HTML-Seite speichern" eine andere, eventuell schon bestehende HTML-Datei als Ziel auswählen. Auf diese Art und Weise ist es sogar möglich, mehrere DynaGeoX-Fenster in eine HTML-Seite einzufügen. Beachten Sie aber, dass Sie in einem solchen Fall selbst dafür verantwortlich sind, dass beim Transport der HTML-Datei in eine andere Umgebung auch alle zugehörigen GEO-Dateien mitgenommen werden! Normalerweise sollten Sie sich auf ein DynaGeoX-Objekt pro HTML-Seite beschränken und die Dateinamen synchron belassen.
