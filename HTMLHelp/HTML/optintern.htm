Einstellungen | Internes

Mausfangbereich:

    Dieser Parameter regelt, wie dicht der Mauszeiger bei einem geometrischen Objekt stehen muss, damit dieses eingefangen wird.
    Bereich [ 1........ (5) ......... 25 ]

Zoomfaktor:

    Dies ist der Streckfaktor, mit dem das Bild zentrisch gestreckt wird, wenn Sie den Menübefehl Bearbeiten | Zeichnung vergrößern anwählen.
    Bereich: [ 1 ...... (1.5) ....... 10 ]

Genauigkeit beim Winkelvergleich:

    Ein Winkel w1 wird einem anderen Winkel w2 als gleich erachtet, wenn w1 im Intervall [w2-dw, w2+dw] liegt. In diesem Feld können Sie dw eingeben, wobei der Wert (ausnahmsweise!) im Gradmaß interpretiert wird, ohne dass ein Gradzeichen nachgestellt wird: die Einheit ist schon hinter dem Eingabefeld vermerkt.
    Bereich: [ 10-5........(0.5)........10 ]

Genauigkeit beim Längenvergleich:

    Zwei Längen d1 und d2 werden als gleich erachtet, wenn d1 im Intervall [d2-dd, d2+dd] liegt. In diesem Feld können Sie dd in der Einheit Pixel eingeben.
    Bereich: [ 1*10-12.......(1*10-8).......1 ]

"Punkt"-Befehl mit erweiterter Funktionalität:

    Wenn Sie diese Option aktivieren, wird der Befehl "Punkt erzeugen" so erweitert, dass er auch einen Punkt auf einer Linie bzw. Schnittpunkte erzeugen kann. Im Einzelnen können folgende Fälle auftreten:

    • Wenn Sie an eine freie Stelle Ihrer Zeichnung klicken, an der es keine Linie gibt, wird ein neuer Basispunkt erzeugt. Das ist die normale Funktion dieses Befehls.
    • Wenn Sie auf eine Stelle klicken, an der es genau eine Linie gibt, wird ein neuer Basispunkt erzeugt und an diese Linie gebunden.
    • Wenn sich an der Stelle, auf die Sie klicken, genau zwei Linien kreuzen, wird dieser Schnittpunkt der beiden Linien erzeugt. Sollten diese beiden Linien jedoch noch mehr Schnittpunkte haben, werden diese ebenfalls erzeugt: DynaGeo erzeugt stets die gesamte Schnittmenge zweier Linien.
    • Wenn durch die angeklickte Stelle drei oder mehr Linien verlaufen, erhalten Sie nur eine Fehlermeldung, weil DynaGeo dann nicht entscheiden kann, was nun sinnvollerweise zu tun wäre.

    Wem eine klare logische Konzeption wichtiger ist als vordergründiger Oberflächen-Komfort, der wird diese Option deaktiviert lassen: nicht umsonst wird in DynaGeo konsequent zwischen Basispunkten und abhängigen Punkten unterschieden!

Korrektheits-Prüfung animiert durchführen:

    Wenn dieses Kästchen markiert ist, wird während einer Korrektheitsprüfung der von DynaGeo intern durchgeführte Zugvorgang auf dem Bildschirm dargestellt. Damit können Sie DynaGeo sozusagen "beim Nachdenken zuschauen". Wenn Sie die Markierung entfernen, wird der Zugvorgang nur intern durchgerechnet. Der Verzicht auf die Darstellung auf dem Bildschirm beschleunigt den Vorgang enorm.

Fette Mauszeiger verwenden:

    Diese Option ist für Sehbehinderte gedacht, die die dünnen Linien der Standard-Auswahl-Cursor von EUKLID DynaGeo nicht gut genug erkennen können. Wenn Sie dieses Kästchen markieren, werden dicker gezeichnete Auswahl-Cursor verwendet.

XML-Ausgabe-Format:

    Hier können Sie interne Details über das Format der GEO-Dateien einstellen. Es stehen 4 Optionen zur Verfügung:

    1. ZIP-komprimiert:
      dies führt zu recht kompakten GEO-Dateien, die allerdings den für XML typischen Vorteil einbüßen, lesbaren Klartext zu enthalten.

    2. unformatiert:
      dies führt zu relativ kleinen GEO-Dateien, die in geeigneten Text-Editoren darstellbar sind. Beim Versenden als EMail-Attachments können solche Dateien jedoch Probleme bereiten, wenn ein EMail-Programm nicht korrekt mit den hierbei verwendeten langen Zeichenketten umgehen kann.

    3. kurze Zeilen:
      dies ist das voreingestellte und empfohlene GEO-Format. Bei der Erstellung der Datei werden Zeilenumbrüche eingefügt, so dass erst gar keine zu langen Zeilen entstehen und somit auch kaum Probleme zu erwarten sind, wenn Sie diese GEO-Dateien als EMail-Attachments versenden. Allerdings sind solche Dateien durch die zusätzlichen Zeichen etwas größer als die jeweils entsprechende "unformatierte" Version.

    4. "pretty print":
      hier werden zusätzlich zu den Zeilenumbrüchen des "Kurze Zeilen"-Formats noch Tabulatorzeichen eingefügt, um die Hierarchie-Stufen innerhalb der XML-Datei zu verdeutlichen. Die Dateien werden dadurch jedoch nochmals deutlich vergrößert, was sich nur lohnt, wenn Sie sie regelmäßig in einem Text-Editor Ihrer Wahl betrachten wollen.

    Im Normalfall sollten Sie das empfohlene Format "Kurze Zeilen" eingestellt lassen, sofern Sie nicht ernsthafte Gründe dafür haben, eine andere Variante zu wählen. Wenn Sie den Inhalt der GEO-Datei trotzdem "schön dargestellt" sehen wollen, laden Sie die Datei einfach in den Internet-Explorer: dieser stellt sogar die von der "unformatierten" Variante gelieferten XML-Daten sehr übersichtlich dar!


Experten-Optionen:

    Dieser Knopf führt Sie (nach einer sehr ernstgemeinten Warnung!) zu einem eigenen Fenster, indem Sie einige nicht ganz ungefährliche interne Parameter von DynaGeo einstellen können:

    1. Im Kasten "Aktualisierungsstrategie im Zugmodus" können Sie die Strategie wählen, nach der im Zugmodus die Lage der jeweils abhängigen Objekte festgelegt wird. Sie entscheiden damit, ob Ihnen die Determiniertheit oder die Kontinuität Ihrer Zeichnung wichtiger ist. Als Standard für jede neu erstellte Zeichnung ist eine reversible Aktualisierungsstrategie eingestellt. Weitere Details zu diesem nicht ganz einfachen Thema finden Sie im Abschnitt über Das Problem der "Springenden Punkte".

      Die vom Autor für seine Zeichnung gewünschte Aktualisierungsstrategie wird in der GEO-Datei mit abgespeichert und bei jedem späteren Öffnen der Zeichnung automatisch wieder eingestellt. Sie sollten diese Einstellung nur ändern, wenn Sie sehr genau wissen, was Sie tun!

      Achtung !

        EUKLID DynaGeo ist von seiner Anlage her ein eher auf Reversibilität ausgerichtetes Geometrie-Programm. Sollte es bei der weiteren Entwicklung zu Situationen kommen, in denen eine Entscheidung zwischen deterministischer und reversibler Aktualisierungsstrategie unausweichlich erscheint, dann wird stets der Reversibilität der Vorrang eingeräumt werden.

    2. Im Kasten "Rekursion" können Sie darüber entscheiden, ob DynaGeo rekursive Funktions-Definitionen zulassen soll oder nicht. Im Normalfall lehnt DynaGeo jegliche Rekursion ab. Weitere Details zu diesem nicht ganz einfachen Thema finden Sie im Abschnitt "Das Problem der rekursiven Funktionen".

      Achtung !

        Die Definition einer rekursiven Funktion ist ein trickreicher und fehleranfälliger Vorgang. In der Standard-Konfiguration ist diese Option in EUKLID DynaGeo abgewählt, so dass das Programm jeden Versuch einer rekursiven Definition ablehnt. Wenn Sie auf Stabilität Wert legen, dann sollten Sie es dabei belassen.
