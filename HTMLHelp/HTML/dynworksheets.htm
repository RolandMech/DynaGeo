Wie stelle ich dynamische Arbeitsblätter her?

Am einfachsten benutzen Sie dazu die Menübefehle Als DynaGeoX-HTML-Seite speichern bzw. Als DynaGeoJ-HTML-Seite speichern. Wenn EUKLID DynaGeo einen dieser Befehle ausführt, werden stets zwei Dateien mit verschiedenen Erweiterungen ins Zielverzeichnis geschrieben:

  • die erste ist eine HTML-Datei, die das Fenster enthält, in welchem die Zeichnung erscheinen soll;
  • die zweite ist eine GEO-Datei, die die Zeichnung enthält, welche in diesem Fenster dargestellt werden soll.

Die Einbindung der dynamischen Zeichnung geschieht nun in dieser HTML-Datei mit Hilfe spezieller "Tags": im Falle von DynaGeoX mit einem <OBJECT>-Tag, im Falle von DynaGeoJ mit einem <APPLET>-Tag. Sie können die HTML-Datei mit einem passenden Editor weiterbearbeiten, müssen aber peinlich darauf achten, dass dieser nicht eigenmächtig das <OBJECT>- bzw. <APPLET>-Tag verändert. Manche HTML-Editoren gehen recht rücksichtslos mit Dingen um, von denen sie nichts verstehen...

Natürlich können Sie stattdessen auch das <OBJECT>- bzw. <APPLET>-Tag in ein anderes HTML-Dokument kopieren. Schließlich lassen sich auch mehrere Zeichnungen in mehreren Fenstern desselben HTML-Dokuments anzeigen, indem Sie einfach mehrere solche Tags in den HTML-Quelltext einfügen. Achten Sie aber in jedem Fall darauf, dass zur Laufzeit auch die jeweils zugehörige GEO-Datei im selben Verzeichnis steht wie die HTML-Datei!

    Obwohl mehrere Viewer-Fenster im selben HTML-Dokument möglich sind, wird dies nicht empfohlen: in manchen Fällen kommt der Browser mit der Verwaltung mehrerer parallel arbeitender Viewer-Instanzen nicht zurecht.




Tipps für den Export von Zeichungen in HTML-Dokumente:

Obwohl der Vorgang durch die entsprechenden Menü-Befehle technisch weitgehend problemlos ist, sollten Sie dabei doch einige Vorsicht walten lassen, damit die erzielten Ergebnisse auch Ihren Erwartungen entsprechen. Unabhängig vom verwendeten Viewer sollten Sie die folgenden Punkte beachten:

  • Wählen Sie die Größe des Viewer-Fensters mit Bedacht. Machen Sie es nicht größer als nötig; in jedem Fall sollte es vollständig im Browser-Fenster dargestellt werden können. Andererseits sollte es mindestens so groß sein, dass möglichst alle Objekte der Zeichnung (zumindest bei der Initialisierung) im Viewer-Fenster sichtbar sind.

  • Wenn Sie Befehlsknöpfe zur Verfügung stellen wollen, sollten Sie deren Anzahl so klein wie möglich halten. Das Viewer-Fenster muss dabei mindestens so hoch sein, dass es alle Knöpfe untereinander darstellen kann.

  • Verwenden Sie möglichst keine Textboxen in der Zeichnung. Diese kosten nur wertvollen Platz im Viewer-Fenster. Begleitende Texte sind im umgebenden HTML-Dokument besser aufgehoben!

DynaGeoX benutzt dieselben Datenstrukturen wie EUKLID DynaGeo; daher können Sie davon ausgehen, dass im Normalfall eine Zeichnung, die sich unter EUKLID DynaGeo erwartungsgemäß verhält, dann auch in DynaGeoX korrekt dargestellt werden wird. Hingegen verlangt der Export für DynaGeoJ noch etwas mehr Vorsicht von Ihnen, weil es sich hier um ein eigenständiges JAVA-Programm handelt, das sich naturgemäß in einigen internen Details wesentlich von EUKLID DynaGeo unterscheidet. Darüberhinaus sind einige wenige der in DynaGeo verfügbaren Arten geometrischer Objekte in DynaGeoJ bisher noch nicht implementiert; dies betrifft derzeit z.B. die Strecken fester Länge.

Bevor Sie nun anfangen, Ihr ultimatives Online-Geometrie-Buch zu schreiben, sollten Sie sich mit Geduld wappnen, denn die verwendeten Technologien haben durchaus ihre Tücken. Auch sollten Sie gelegentlich BackUps Ihrer Daten machen. Das hat noch nie großen Schaden angerichtet, aber schon so manche Katastrophe verhütet.
