Hinweise für die Installation in Netzen

Die folgenden Ausführungen richten sich vor allem an diejenigen, die EUKLID DynaGeo in einem Netzwerk installieren wollen (oder müssen).


Grundsätzliches:

    EUKLID DynaGeo sollte in Rechnernetzen unbedingt auf dem Server installiert werden. Auch weniger routinierte Administratoren sollten der Versuchung widerstehen, das Programm an jeder Arbeitsstation lokal zu installieren! Eine korrekte Netzwerk-Installation von EUKLID DynaGeo ist insgesamt weit weniger aufwendig als die "lokale Variante" und vor allem viel einfacher zu pflegen!
    Zur Installation von EUKLID DynaGeo für den Netzbetrieb sind folgende Arbeitsschritte nötig:

    • Loggen Sie sich an einer Arbeitsstation als Administrator ein.
    • Starten Sie das Installationsarchiv EDG*.EXE und
    • installieren EUKLID DynaGeo in ein Verzeichnis Ihrer Wahl auf Ihrem Server.
    • Erstellen Sie an allen Client-Rechnern eine Verknüpfung zur DYNAGEO.EXE auf dem Server.
    • Geben Sie Ihren Benutzern im DynaGeo-Verzeichnis Lese-Rechte.


Erläuterungen:

    Der erste Unterschied zu einer lokalen Installation ist, dass Sie sich als Administrator anmelden müssen, damit Sie nämlich in das Zielverzeichnis der Installation ("DynaGeo-Verzeichnis") auf Ihrem Server überhaupt schreiben dürfen. Starten Sie dann das Installationsarchiv.

    Es macht wenig Sinn, mehr als das "normale" EUKLID DynaGeo-Programm auf dem Server zu installieren. Weder die Installation der Viewer noch die Registrierung der DynaGeo-Datei-Typen auf dem Server bringt Ihnen einen wesentlichen Nutzen an den Clients in Ihrem Netz. Wählen Sie daher im entsprechenden Auswahldialog des Installations-Programms alle "zusätzlichen Aufgaben" ab!

    Wenn das Installationsprogramm seine Arbeit getan hat, ruft es die neu erstellte DYNAGEO.EXE aus dem DynaGeo-Verzeichnis auf. Dieser erste Start von EUKLID DynaGeo muss unbedingt mit Schreibrechten im DynaGeo-Verzeichnis geschehen, was durch Ihre Anmeldung als Administrator ja gewährleistet ist. Nun wird im DynaGeo-Verzeichnis die globale Konfigurationsdatei DYNAGEO.INI erstellt. Zur Ermittlung der Lizenzdaten sucht das Programm dann nach einer *.DGL-Datei, und zwar in dem Verzeichnis, in dem auch das Installationsarchiv steht. Wird eine DGL-Datei gefunden, dann wird eine Kopie dieser Datei ins DynaGeo-Verzeichnis geschrieben. (Wird keine DGL-Datei gefunden, ist die installierte Version Freeware - und der Unterrichtseinsatz damit erlaubt!) Damit ist EUKLID DynaGeo auf dem Server schon fertig installiert.

    An der Arbeitsstation, von der aus Sie die Installation durchführen, wird darüberhinaus im Startmenü eine Verknüpfung zur DYNAGEO.EXE auf dem Server erstellt, mit der Sie das Programm an dieser Arbeitsstation auch schon starten können. Damit das auch an den anderen Client-Rechnern Ihres Netzes funktioniert, müssen Sie an jedem dieser Rechner nur noch diese Verknüpfung zur DYNAGEO.EXE auf dem Server erstellen, je nach Geschmack auf dem Desktop oder im Startmenü. Mehr ist nicht nötig! EUKLID DynaGeo braucht an den Client-Rechnern keine DLLs im lokalen Windows-Verzeichnis und auch keine Einträge in der Registry.

    Damit nun auch Ihre Benutzer in den Genuss des neuinstallierten EUKLID DynaGeo kommen, müssen Sie ihnen noch die Rechte zum Lesen, Suchen und Ausführen von Dateien im DynaGeo-Verzeichnis auf dem Server geben. Diese Rechte sollten auch für die Unterverzeichnisse des DynaGeo-Verzeichnisses gelten, damit jeder Benutzer auch Zugriff auf die Beispieldateien hat. Zur Gewährleistung der Betriebssicherheit sollten die "gewöhnlichen Benutzer" aber keinerlei Schreibrechte im DynaGeo-Verzeichnis und seinen Unterverzeichnissen haben!

    Damit ist die Installation abgeschlossen, und EUKLID DynaGeo sollte allen Benutzern, denen Sie den Zugriff erlauben, zur Verfügung stehen. Der Aufwand einer Netzwerk-Installation ist also wirklich weit geringer als der, den Sie treiben müssten, um das Programm lokal an allen Arbeitsstationen zu installieren. Und beim nächsten Update brauchen Sie nur noch das DynaGeo-Verzeichnis auf dem Server zu aktualisieren; an den Client-Rechnern sind keine weiteren Nacharbeiten mehr nötig!


Technische Interna:

    Für einen normalen Programmstart werden die 3 Dateien DYNAGEO.EXE, DYNAGEO.CHM und DYNAGEO.INI gebraucht. Diese drei Dateien müssen zusammen in einem beliebigen Verzeichnis stehen, das im folgenden DynaGeo-Verzeichnis genannt wird. Jeder EUKLID DynaGeo-Benutzer muss auf diese 3 Dateien mindestens lesenden Zugriff haben. Üblicherweise sollten normale Benutzer im DynaGeo-Verzeichnis keinerlei Schreibrechte haben; so sind die Originaldateien sicher aufbewahrt. Sie sollten jedoch DYNAGEO.EXE und vor allem DYNAGEO.CHM gegebenenfalls als "shareable" deklarieren, damit viele User parallelen Zugriff haben.

    Damit ein Benutzer seine privaten Einstellungen auch abspeichern kann, braucht DynaGeo eine weitere Konfigurationsdatei namens EDG_USER.INI, für die der Benutzer Schreibrechte hat. Diese Datei befindet sich bei älteren Windows-Versionen (95, 98, ME) ebenfalls im DynaGeo-Verzeichnis; bei den moderneren NT-Varianten (NT4, 2000, XP,...) wird sie ins Profil des aktuellen Benutzers geschrieben, und zwar ins Verzeichnis %appdata%\dynageo\. Sie brauchen sich um die Erstellung dieser Datei nicht zu kümmern - DynaGeo macht das automatisch! Beim Start werden die zuletzt abgespeicherten Benutzer-Einstellungen aus dieser Datei ausgelesen. Am Ende der Arbeitssitzung werden die nun aktuellen Einstellungen wiederum dort abgespeichert. Bei jedem Start wird sich das Programm bemühen, möglichst genau dieselben Verhältnisse wieder herzustellen, wie sie am Ende der vorigen Arbeitssitzung herrschten.

    Wo die Daten (also die *.GEO- und *.MAK-Dateien) stehen, ist für EUKLID DynaGeo völlig egal. Ein Benutzer kann innerhalb des Programms auf alle ihm im Netz bekannten Laufwerke und Verzeichnisse zugreifen.


    Achtung !

      EUKLID DynaGeo braucht für den normalen Betrieb weder irgendwelche DLLs im lokalen Windows-Verzeichnis noch irgendwelche Einträge in der Registry.

      Wenn Sie jedoch den DynaGeoX-Viewer einsetzen wollen, um dynamischer Zeichnungen in HTML-Dokumenten darzustellen, dann sind dazu einige Einträge in die lokale Registry des Client-Rechners bzw. des jeweiligen Benutzers nötig; diese werden aber beim ersten Aufruf von DynaGeoX automatisch gesetzt - sofern Sie als Administrator das in Ihrem Netz erlaubt haben! Ebenso werden natürlich die Informationen über die mit EUKLID DynaGeo assoziierten Datei-Typen (*.geo und eventuell das veraltete *.geox) in der Registry vermerkt. Und für den Fall, dass Sie das Programm wieder von einem Rechner entfernen wollen, werden De-Installations-Daten in der Registrierungsdatenbank hinterlegt - obwohl ich mir kaum vorstellen kann, dass Sie EUKLID DynaGeo jemals wieder deinstallieren wollen... ;-)
