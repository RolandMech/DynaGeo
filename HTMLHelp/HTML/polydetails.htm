Details zu Polygonen

Im Gegensatz zu früheren Versionen von EUKLID DynaGeo, in denen ein Polygon als Fläche angesehen wurde, wird es nun als geschlossener Streckenzug definiert: das Polygon ist der Rand der Polygonfläche!

Wenn Sie ein Polygon (mit dem Befehl Dreieck oder N-Eck) erzeugen, werden zusätzlich auch seine Seitenstrecken (als individuell referenzierbare Objekte) erzeugt. Dadurch wird eine eigene Darstellung für das Polygon überflüssig. Es ist somit für den Benutzer eigentlich unsichtbar und hat eher den virtuellen Charakter einer "Verwaltungseinheit" als den eines (im engeren Sinn) geometrischen Objekts. Auch wenn Sie alle Seiten(strecken) eines Polygons löschen, ist das Polygon selbst nach wie vor in Ihrer Zeichnung vorhanden: Sie können auch dann noch einen Punkt an dieses Polygon binden oder die von ihm umschlossene Fläche berechnen!

Wollen Sie ein Polygon gefüllt darstellen, dann können Sie dies jederzeit nachträglich mit dem Befehl Fläche füllen erreichen. Wollen Sie grundsätzlich nur gefüllte Polygone erstellen, können Sie in den Einstellungen auf der Registerseite Darstellung die Option "Polygone beim Erzeugen automatisch füllen" aktivieren.

Obwohl die Polygone über keine eigene visuelle Darstellung verfügen, können sie mit der Maus angewählt werden, und man kann ihr Kontextmenü aufrufen. In diesem gibt es neben den üblichen Einträgen ("Verbergen", "Benennen...") auch den Menüpunkt "Farbe...". Wenn Sie hier eine neue Farbe setzen, werden alle Seiten des Polygons in dieser Farbe dargestellt. Ist eine Füllung vorhanden, dann wird diese mit einer aufgehellten Variante der neuen Farbe versehen. Sie können aber auch jede einzelne Polygonseite sowie die Füllung mit einer jeweils eigenen Farbe einfärben, indem Sie die für das jeweilige Teilobjekt gewünschte Farbe über dessen eigenes Kontextmenü individuell einstellen.
